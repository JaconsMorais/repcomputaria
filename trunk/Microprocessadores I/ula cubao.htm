a. Descrição do Problema:

Uma  Unidade Lógica Aritmética (ALU) é um circuito combinacional que realiza micro-operações lógica e aritmética em um par de operandos de n-bit (ex. A[3:0] e B[3:0]). As operações realizadas por uma ULA são controladas por um conjunto de entradas de seleção de funções. Neste experimento, você irá projetar uma ULA de 4-bit ALU com 3 entradas de seleção de funções: Modo M, Entradas Seleção S1 e S0. A entrada modo M seleciona entre uma operação Lógica (M=0) e Aritmética (M=1). As funções realizadas pela ULA estão especificadas na Tabela I.

Ao fazer uma operação aritmética, você precisa decidir como representar números negativos. Geralmente em sistemas digitais, números negativos são representados em complemento de dois. Isto tem várias vantagens em relação a representação sinal e magnitude como a facilidade de realizar operações de adição e subtração de números negativos e positivos. Além disto, o número zero tem uma representação única em complemento de dois. O complemento de dois de um número N de n-bit é definida como,
 

2n - N = (2n - 1 - N) + 1
A última representação fornece uma forma fácil de encontrar o complemento de dois: pegue o complemento do número e adicione 1. Por exemplo, para representar o número -5, nós calculamos o complemento de dois de 5 (=0101) da seguinte forma,
   5   0 1 0 1   -->      1 0 1 0  (complemento bit a bit)
                              + 1
                          1 0 1 1  (complemento de dois)

Números representados em complemento de dois estão na faixa -(2n-1) a  +(2n-1 - 1). Para um número de 4-bit isto significa que o número está na faixa de -8 a +7. Existe um problema potencial que nós precisamos ficar atentos quando trabalhamos com complemento de dois, ou seja, overflow (resultado acima do limite superior da faixa) e underflow(resultado abaixo do limite inferior da faixa) como ilustrado no exemplo abaixo, 

               0 1 0 0    (=carry Ci)
        +5       0 1 0 1
        +4   +   0 1 0 0
        +9     0 1 0 0 1    = -7!


também, 

               1 0 0 0    (=carry Ci)
       -7        1 0 0 1
       -2     +  1 1 1 0
       -9      1 0 1 1 1    = +7!

Ambos os cálculos dão resultados errados (-7 ao invés de +9 ou +7 ao invés de -9) pelo fato de que o resultado +9 ou -9 está fora da faixa permitida para números em complemento de dois de 4-bit. Sempre que o resultado for maior do que +7 ou menor do que -8 existe um overflow ou underflow e o resultado da adição ou subtração está errado. Overflow e underflow podem ser facilmente detectados quando o bit carry-out do estágio mais significativo (ou seja, C4 ) for diferente do carry-out do estágio anterior (ou seja, C3).
Você pode assumir que as entradas A e B estão em complemento de dois quando eles são apresentados à entrada da ULA.

b. Estratégias de Projeto

Quando você for projetar a ULA, você deve seguir o princípio "Dividir e Conquistar" para desenvolver um projeto modular que consiste de blocos menores e mais fáceis de entender, alguns podem ser reutilizados. Ao invés de projetar uma ULA de 4-bit como um único circuito, nós iremos primeiro projetar uma ULA de um-bit, também chamada de bit-slice. Estes bit-slices podem ser colocados juntos para fazer uma ULA de 4-bit.

Existem diferentes formas de projetar um bit-slice da ULA. Um método consiste em escrever a tabela-verdade para a ULA de um-bit. Esta tabela tem 6 entradas (M, S1, S0, C0, Ai e Bi) e duas saídas Fi e Ci+1. Isto pode ser feito mas pode ser tediosos quando feito manualmente.

Uma forma alternativa é dividir a ULA em dois módulos, um Lógico e um Aritmético. Projetar cada módulo separadamente será mais fácil do que projetar um bit-slice como uma única unidade. Um diagrama de bloco possível da ULA é mostrado na Figura 2. Ele consiste de três módulos: 2:1 MUX, uma unidade Lógica e uma unidade Aritmética.

B. Experimentos
Sua tarefa é projetar e implementar a ULA de 4-bit usando as ferramentas do Xilinx Foundation. Siga as instruções para o projeto da ULA de 4-bit. Você irá criar um projeto (veja as tarefas abaixo) com um esquemático de alto nível que consiste de uma ULA de 4-bit. Este arquivo de alto nível terá várias macros que você precisa criar usando o editor de Esquemático ou VHDL. Macros podem ter macros inseridas dentro delas.

Como este projeto é mais complicado do que os projetos anteriores, será importante que você implemene o projeto de forma sistemática. Cada macro deve ser simulada e erros devem ser corrigidos antes de passar para a próxima etapa. Se você não fizer isto, será mias difícil encontrar erros no sistema

Tarefa

  • Criar um novo projeto no modo esquemático: Chamá-lo MY4ALU, onde MY é substituído pelo seu nome.
  • Abra o editor de esquemático (a folha deve ter o mesmo nome do projeto: MY4ALU1.sch). Este será o seu esquemático de alto nível. (Note: Cda vez que você abre uma nova folha, ela será chamada MY4ALU2.sch, MY4ALU3.sch; todas estas folhas são parte do esquemático de alto nível. Você deve usar múltiplas folhas apenas quando o seu esquemático de alto nível é tão grande que não cabe em uma única folha).
  • Criar macros para cada módulo (cuidadosamente leia a seção "Creating Macros" nos Tutoriais da Xilinx - não use a folha de mais alto nível como uma macro!). Isto pode ser feito na janela do Editor de Esquemático selecionando TOOLS -> SYMBOL WIZARD.
  • Defina os arquivos fontes para as macros.
  • Faça uma simulação funcional das macros simulation of  the macros (primeiro coloque-as no esquemático de mais alto nível). Verifique a operação correta antes de avançar para o próximo passo. Isto irá tornar a correção dos erros mais fácil. Se você encontrar erros nas formas de onda simuladas, procure o erro no circuito. Checar o seu esquemático ou código HDL. Lembre que nomes de macros HDL devem ter no máximo 8 caracteres. Ao checar o esquemático, use a ponta de prova (seta - Probe) para adicionar sinais (nomes dos fios - net names) à janela do simulador. Isto irá ajudá-lo a rastrear o erro de uma forma sistemática.
  • Faça o bit slice (isto deve ser uma macro contendo várias sub macros)
  • Faça a ULA de 4-bit.
  • Projete e adicione o circuito que detecta overflow V ao seu esquemático.
  • Simule a ULA de 4-bit e verifique se todas as operações lógicas e aritméticas funcionam corretamente.
  • Será instrutivo fazer uma simulação funcional para ter uma idéia da velocidade do circuito.

Se o seu projeto for muito grande você pode facilmente compactá-lo através da janela Project Manager e selecionando FILE-> ARCHIVE PROJECT. Isto irá compactar todo o projeto incluindo todas as bibliotecas necessárias.
